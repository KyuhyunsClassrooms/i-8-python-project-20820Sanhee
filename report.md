# 최종 보고서

## 1. 프로그램 주제와 목적

내 프로그램의 주제: 나만의 스마트 공부 습관 점수 분석기

이 프로그램을 만든 이유:하루 공부 기록을 입력하면 과목별 효율과 전체 공부 습관을 점수로 분석해 주기 위해 만들었다. 공부 계획 대비 실천 정도와 집중도를 함께 반영하여 자신의 공부 습관을 객관적으로 확인하고 개선하도록 돕는 것이 목적이다.


---

## 2. 문제 분해 및 모델링

### 입력

입력
과목 수
과목명
계획 공부 시간
실제 공부 시간
집중도(1~5)
### 처리
처리
달성률 계산: 실제 시간 / 계획 시간 × 100
효율 점수 계산: (달성률 × 0.6) + (집중도 × 8)
계획했지만 실제 공부 시간이 0이면 0점 처리
최대 점수는 100점으로 제한
평균 점수 계산 및 최고 효율 과목 찾기

### 출력
출력
과목별 달성률 및 효율 점수
전체 평균 점수
공부 등급(A~F)
최고 효율 과목
내일의 공부 조언

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름: study_logs

한 행의 의미: 한 과목의 공부 기록 (과목명, 계획 시간, 실제 시간, 집중도)

각 열의 의미:과목명, 계획 시간, 실제 시간, 집중도 같은 서로 다른 형태의 연관된 데이터를 한 행으로 묶어 체계적으로 관리하기 위해서입니다. 또한, 여러 과목의 데이터를 일반 리스트 여러 개로 나누어 만드는 것보다, 2차원 리스트를 사용하면 반복문 하나로 모든 과목의 분석 점수를 일괄 계산하기 훨씬 편리하기 때문에 선택했습니다.

| 열 번호 | 의미          | 실제 예시 |
| ---: | ----------- | ----- |
| 0번 열 | 과목명         | 수학    |
| 1번 열 | 계획 공부 시간(분) | 120   |
| 2번 열 | 실제 공부 시간(분) | 90    |
| 3번 열 | 집중도(1~5)    | 4     |


이 2차원 리스트를 사용한 이유:

여러 과목의 공부 정보를 한 번에 저장하고 반복문으로 쉽게 분석하기 위해 사용하였다.

## 4. 함수 설명

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
| input_log() |  사용자에게 과목 수와 과목별 정보를 입력받아 데이터 유효성 검사를 거쳐 2차원 리스트를 생성함| 없음 | 생성된 공부 기록 2차원 리스트 (study_logs) |
| analyze_efficiency() | 2차원 리스트를 순회하며 효율 점수를 계산하고, 최댓값 알고리즘으로 최고 효율 과목을 찾아냄 | 공부 기록 2차원 리스트 (logs) | 종합 평균 점수 (average_score), 최고 효율 과목명 (best_subject) |
| show_report() | 분석된 결과를 바탕으로 최종 공부 등급(A~F)을 판정하고 맞춤형 한 줄 조언을 화면에 출력함 | 평균 점수 (avg_score), 최고 효율 과목명 (best_subject) | 없음 (콘솔 화면에 결과 출력) |

가장 중요한 함수 하나를 골라 자세히 설명하세요.

함수 이름:analyze_efficiency(logs)

설명:

설명: 이 함수는 2차원 리스트에 저장된 과목별 공부 데이터를 for 반복문으로 하나씩 꺼내어 산희 학생이 직접 만든 공식으로 효율 점수를 계산하고, if 조건문을 활용한 최댓값 알고리즘으로 오늘 가장 효율적으로 공부한 과목을 찾아내어 종합 평균 점수와 함께 반환(return)하는 핵심 처리 함수입니다.

## 5. 조건문과 반복문 활용

### 조건문 사용 부분

과목 수가 0 이하일 때 다시 입력
집중도가 1~5 범위를 벗어나면 재입력
계획했지만 실제 공부 시간이 0이면 0점 처리
점수가 100점을 넘으면 100으로 제한
등급(A~F) 결정


### 반복문 사용 부분

과목 수만큼 입력 반복
집중도 유효값 입력될 때까지 반복
모든 과목을 돌면서 점수 계산 반복


### 조건문과 반복문이 함께 작동한 부분


---과목 입력 반복 + 집중도 검사 while문
각 과목을 반복하면서 점수 조건 처리
